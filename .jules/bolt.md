## 2024-04-23 - Avoid in-memory filtering for time-bound queries
**Learning:** In backend routes, specifically `dashboard_user_growth`, fetching all rows (`.all()`) and filtering them in memory by Python loop can cause significant memory and CPU overhead as the number of users grows. The current implementation fetches all users regardless of `created_at` timestamp and then calculates buckets in Python.
**Action:** When querying the database for aggregations bounded by time (e.g. `start_date`), apply time filters directly within the SQLAlchemy query (`.filter(User.created_at >= start_date)`) and ideally perform aggregations in the database using `group_by`. Since database engines vary, fetching only the required timeframe and counting in python is also a valid improvement if db-level grouping by date is complex across dialects, but in this application SQLAlchemy cast to Date is supported.
